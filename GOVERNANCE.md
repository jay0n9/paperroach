# Governance

PaperRoach is currently maintainer-led. The maintainer is responsible for
release decisions, compatibility policy, repository settings, and enforcing
the Code of Conduct.

Contributors can propose changes through issues and pull requests. Small,
well-tested fixes may be merged directly after review. Changes affecting the
note schema, data identity, taxonomy, vector-store compatibility, privacy, or
the public CLI should first have an issue describing alternatives and migration
impact.

As the maintainer group grows, this document will be updated to describe
reviewers, voting, release authority, and succession.
